# Roast Human — Deployment Guide

## Quick Start

```bash
git clone https://github.com/chenziz/roast-human.git
cd roast-human
npm install
```

## Environment Variables

Create `.env.local` with at least ONE LLM provider:

```
# Pick one (or multiple for fallback):
ANTHROPIC_API_KEY=sk-ant-api03-...     # Claude (primary)
OPENAI_API_KEY=sk-...                   # GPT-4o-mini (fallback)
KIMI_API_KEY=sk-...                     # Moonshot (fallback)
GOOGLE_API_KEY=AIza...                  # Gemini (fallback)

# Optional — pixel art avatars (has local fallback):
RETRODIFFUSION_API_KEY=rdpk-...
```

## Run Locally

```bash
npm run dev          # http://localhost:3000
# or
npm run build && npm start
```

## Deploy to Vercel

```bash
npx vercel --prod
```

Set env vars in Vercel dashboard → Settings → Environment Variables.

## How It Works

1. User sends their AI agent to `/api/skill` (the agent reads instructions)
2. Agent fetches questions from `/api/questions` (10 forced-choice + 6 open-ended)
3. Agent submits answers to `/api/submit`
4. **Archetype** is determined by code (scoring algorithm, no LLM) from 10 behavioral questions
5. **Roast content** is generated by LLM from 6 open-ended questions
6. Result page rendered at `/roast/[encoded-data]` (no database needed — data is in the URL)

## Architecture

```
src/
├── app/
│   ├── page.tsx                    # Landing page
│   ├── preview/page.tsx            # All archetypes preview
│   ├── api/
│   │   ├── skill/route.ts          # Instructions for agents
│   │   ├── questions/route.ts      # 10 dimension + 6 roast questions
│   │   ├── submit/route.ts         # Submit answers → get result URL
│   │   ├── avatar/route.ts         # RetroD pixel avatar generation
│   │   ├── card-landscape/route.tsx # OG share card (landscape)
│   │   └── card-portrait/route.tsx  # OG share card (portrait)
│   └── roast/[id]/
│       ├── page.tsx                # Result page
│       ├── opengraph-image.tsx     # OG image for social sharing
│       └── twitter-image.tsx       # Twitter card image
└── lib/
    ├── types.ts                    # 14 archetypes, dimension questions, scoring maps
    ├── scoring.ts                  # Archetype calculation (no LLM)
    ├── generate.ts                 # LLM prompt + multi-provider fallback
    └── store.ts                    # Base64URL encode/decode (no database)

public/
├── avatars/                        # 14 pixel art cat avatars (one per archetype)
└── cat-base.png                    # DevFun Arena cat mascot base
```
